# AWS Certified Cloud Practitioner (CLF-C02) 基礎研究

2024年度前半、LinuCの習得と並行して進めたAWSクラウド基盤の全体像把握。
単なる用語暗記ではなく、インフラエンジニアの視点からAWSの「共有責任モデル」と「疎結合」の概念を実機検証した記録。

## 1. クラウド概念とグローバルインフラ
- **リージョンとAZ**: 高可用性を実現するための物理・論理構成の理解。
- **共有責任モデル**: AWS側（クラウドのセキュリティ）とユーザー側（クラウド内のセキュリティ）の境界線の明確化。
- **疎結合の実現**: SQSやELBを活用した、障害耐性の高い分散システムの基礎設計。

## 2. コアサービスの実践的理解
- **Compute**: EC2 インスタンスタイプ（T系/M系）の特性と、Lambdaによるサーバーレスの初歩。
- **Storage**: S3のバケットポリシー、ライフサイクルルール、およびEBSのボリュームタイプの使い分け。
- **Network**: VPC、サブネット、ルートテーブル、インターネットゲートウェイの基本トラフィックフロー。

## 3. コスト・セキュリティ・サポート
- **AWS Cost Explorer**: 予算アラートの設定と、コスト最適化の基本（リザーブドインスタンスの概念）。
- **AWS Trusted Advisor**: 推奨事項に基づいた「コスト削減」「セキュリティ向上」のベストプラクティス検証。
- **Identity and Access Management (IAM)**: ルートアカウントの保護、および最小権限のIAMポリシー設計。
